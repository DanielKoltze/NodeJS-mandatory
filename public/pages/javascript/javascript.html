Callback functions

First thing we will cover is callback functions because these will be essential to know for some of the other topics.

A callback function is a function that is passed as an argument to another function and is executed after some specific. In other words, instead of passing data as an argument to a function, you're passing a function that can perform some behavior or operation on that data.

Here is a small example:

function doSomething(callback) {

  console.log("something...")

  callback()

}

function helloWorld() {

  console.log("Hello world!")

}

doSomething(sayHello)

In this example we define a function called doSomething() that takes a single argument, which is expected to be a callback function. Inside the doSomething() function we perform some task and then call the callback function.

We call the doSomething() function and pass the sayHello() function as the callback. When doSomething() is executed, it logs the message "Doing something..." to the console and then calls the sayHello() function.


Another example of a callback function is in the setInterval() or/and setTimerout(). This could look something like this:

setInterval( () => {

  console.log("wait 1 second to excute")

},1000)


Destructuring

Destructuring is a feature in JavaScript that allows you to extract values from arrays or objects and assign them to variables. You can extract values from an array by using square brackets [] or from an object by using curly braces {}.

Here are 2 examples:

const arr = [1, 2, 3]

const [a, b, c] = arr


const obj = { name: 'Daniel', age: 22 }

const { name, age } = obj

Fetch and promises

The fetch() method is used to send HTTP requests to a server and receive responses from it. It takes 2 arguments:

  • First argument is the url. A string representing the URL to which the request is sent.

  • The second argument is an object containing request method, headers, body, etc. The method attribute is by default set to a GET.

The fetch() method returns a promise which an object that represents a value that may not be available yet. Promises are widely used to handle asynchronous operations such as fetches. When an asynchronous operation is started, a promise is returned immediately. This Promise is initially in the pending state, and will transition to be fulfilled or rejected when the operation completes. To handle these promises when can either use .then() and catch() or we can use await/async
